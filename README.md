# Function Visualizer

Десктопное приложение для визуализации тестовых функций оптимизации и работы
алгоритма дифференциальной эволюции (DE). Позволяет строить тепловые карты
двумерных срезов многомерных функций, накладывать изолинии, задавать
ограничения в виде формул и наблюдать эволюцию популяции агентов в реальном
времени.

## Возможности

- Библиотека стандартных бенчмарк-функций: Sphere, Rastrigin, Rosenbrock,
  Ackley, Griewank, Schwefel, Styblinski–Tang, Levy и другие — а также
  произвольная пользовательская формула через парсер выражений.
- Размерность задачи от 2 до 30. Для размерности выше двух визуализируется
  двумерный срез по выбранным осям, остальные координаты фиксируются.
- Тепловая карта на сетке с настраиваемой детализацией по X и Y, палитры
  Viridis и Grayscale.
- Изолинии на основе Marching Squares с корректной обработкой седловых
  ячеек через сравнение среднего значения ячейки с уровнем.
- Ограничения в виде формул (`x1^2 + x2^2 <= 4`, `x1 + x2 >= 1`,
  `x1 * x2 = 0`). Каждое ограничение редактируется отдельной строкой.
  Недопустимая область маскируется поверх heatmap.
- Алгоритм DE/rand/1/bin с настройкой размера популяции, числа итераций,
  параметров F и CR. Обработка ограничений методом квадратичного штрафа.
- Анимированное воспроизведение истории итераций: слайдер, кнопка play,
  треки агентов, подсветка лучшего решения.
- Минимапа с полным доменом функции и индикацией текущего view-окна.
- Отображение координат и значения функции при наведении курсора.
- Drag и zoom по heatmap для навигации.
- Экспорт изображения в PNG, полной истории DE в JSON.

## Технологии

- Java 17
- JavaFX 21
- Gradle (wrapper включён)
- exp4j — парсинг пользовательских математических выражений
- Jackson Databind — сериализация истории DE
- JUnit 5 — модульные тесты

## Сборка и запуск

Требования: JDK 17 или новее. macOS / Windows / Linux.

```bash
./gradlew run
```

Запуск тестов:

```bash
./gradlew test
```

Сборка дистрибутива:

```bash
./gradlew installDist
# результат в build/install/function-visualizer
```

## Архитектура вкратце

Вычисление значений функции на сетке (`GridComputer`) отделено от рендера
(`HeatmapRenderer`, `MarchingSquares`). Полный результат UI-запроса
собирается в `ComputeService` как составной `HeatmapComputationResult` и
применяется к канвасам через `Platform.runLater`.

Алгоритм DE инкапсулирован в `DifferentialEvolutionOptimizer` и реализует
общий интерфейс `PopulationOptimizer`. Задача оптимизации
(`OptimizationProblem`) содержит функцию, границы и список ограничений;
обработка ограничений вынесена в стратегию (`ConstraintHandlingStrategy`
с реализацией `QuadraticPenaltyStrategy`). История итераций накапливается
в `OptimizationHistoryBuilder` и может быть сохранена в JSON через
`HistoryJsonExporter`.

Тяжёлые вычисления (построение сетки, прогон DE) выполняются в отдельном
пуле потоков. UI асинхронен через `CompletableFuture` и дебаунсер, чтобы
не блокироваться на частых изменениях параметров.
